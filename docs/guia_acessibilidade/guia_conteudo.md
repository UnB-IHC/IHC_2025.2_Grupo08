# Checklist de Conteúdo

- [ ] **Usar linguagem simples**, clara e direta, evitando jargões ou frases longas.  
  `WCAG 3.1.5 (AAA)` [(1)](https://www.w3.org/WAI/WCAG22/Understanding/reading-level)

- [ ] **Organizar o conteúdo com títulos, subtítulos e parágrafos curtos**, facilitando navegação e leitura.  
  `WCAG 2.4.6 (AA)` [(2)](https://www.w3.org/WAI/WCAG22/Understanding/headings-and-labels)

- [ ] **Garantir que os títulos descrevam corretamente o conteúdo da seção**.  
  `WCAG 2.4.6 (AA)` [(3)](https://www.w3.org/WAI/WCAG22/Understanding/headings-and-labels)

- [ ] **Incluir textos alternativos (`alt`) adequados para imagens**, explicando função e contexto.  
  `WCAG 1.1.1 (A)` [(4)](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content)

- [ ] **Usar `alt=""` em imagens decorativas**, que não transmitem informação.  
  `WCAG 1.1.1 (A)` [(5)](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content)

- [ ] **Descrever no texto imagens que contêm texto**, garantindo acesso a todas as informações.  
  `WCAG 1.4.5 (AA)` [(6)](https://www.w3.org/WAI/WCAG22/Understanding/images-of-text)

- [ ] **Evitar texto dentro de imagens**, exceto quando essencial (logotipos, gráficos).  
  `WCAG 1.4.5 (AA)` [(7)](https://www.w3.org/WAI/WCAG22/Understanding/images-of-text)

- [ ] **Identificar corretamente a linguagem do documento** usando `lang="pt-BR"` no HTML.  
  `WCAG 3.1.1 (A)` [(8)](https://www.w3.org/WAI/WCAG22/Understanding/language-of-page)

- [ ] **Sinalizar mudança de idioma dentro do texto** (ex.: `<span lang="en">`).  
  `WCAG 3.1.2 (AA)` [(9)](https://www.w3.org/WAI/WCAG22/Understanding/language-of-parts)

- [ ] **Escrever links com textos significativos**, evitando “clique aqui” ou “saiba mais”.  
  `WCAG 2.4.4 (A)` [(10)](https://www.w3.org/WAI/WCAG22/Understanding/link-purpose-in-context)

- [ ] **Evitar abreviações sem explicação**, siglas desconhecidas ou termos muito técnicos.  
  `WCAG 3.1.3 (AAA)` [(11)](https://www.w3.org/WAI/WCAG22/Understanding/unusual-words)

- [ ] **Garantir que listas tenham estrutura de lista** (`<ul>`, `<ol>`, `<li>`), facilitando leitura por tecnologias assistivas.  
  `WCAG 1.3.1 (A)` [(12)](https://www.w3.org/WAI/WCAG22/Understanding/info-and-relationships)

- [ ] **Fornecer transcrições para áudios** e **legendas sincronizadas para vídeos**.  
  `WCAG 1.2.2 (A)` e `1.2.4 (AA)` [(13)](https://www.w3.org/WAI/WCAG22/Understanding/captions-prerecorded)

- [ ] **Garantir que ícones tenham rótulos acessíveis**, seja visíveis ou via `aria-label`.  
  `WCAG 4.1.2 (A)` [(14)](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value)

- [ ] **Evitar uso de metáforas visuais complexas** que possam gerar confusão.  
  *(Guia-UK)* [(15)](#)

- [ ] **Manter consistência terminológica** (nomes de botões, seções, funções idênticas).  
  `(Boa prática)` [(16)](#)

- [ ] **Escrever conteúdos que possam ser compreendidos fora de ordem**, facilitando leitura por leitores de tela.  
  `(Apoiando WCAG 1.3.2)` [(17)](https://www.w3.org/WAI/WCAG22/Understanding/meaningful-sequence)

- [ ] **Garantir que mensagens de erro sejam claras, específicas e orientem a solução.**  
  `WCAG 3.3.1 (A)` [(18)](https://www.w3.org/WAI/WCAG22/Understanding/error-identification)
