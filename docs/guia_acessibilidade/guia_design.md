# Checklist de Design

- [ ] **Garantir indicador de foco visível** em todos os elementos interativos.  
  `WCAG 2.4.7 (AA)` [(1)](https://www.w3.org/WAI/WCAG22/Understanding/focus-visible)

- [ ] **Oferecer mensagens de erro descritivas**, indicando claramente o que está errado e como corrigir.  
  `WCAG 3.3.1 (A)` [(2)](https://www.w3.org/WAI/WCAG22/Understanding/error-identification)

- [ ] **Sugerir correções** sempre que possível em formulários.  
  `WCAG 3.3.3 (AA)` [(3)](https://www.w3.org/WAI/WCAG22/Understanding/error-suggestion)

- [ ] **Garantir contraste adequado de texto**, mínimo 4.5:1.  
  `WCAG 1.4.3 (AA)` [(4)](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum)

- [ ] **Garantir contraste de componentes**, como botões, campos, links e bordas (mínimo 3:1).  
  `WCAG 1.4.11 (AA)` [(5)](https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast)

- [ ] **Assegurar contraste adequado em objetos gráficos** (ícones, elementos informativos, gráficos).  
  `WCAG 1.4.11 (AA)` [(6)](https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast)

- [ ] **Evitar depender apenas de cores** para transmitir informações.  
  `WCAG 1.4.1 (A)` [(7)](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color)

- [ ] **Garantir que links sejam distinguíveis visualmente**, não apenas pela cor.  
  `WCAG 1.4.1 (A)` [(8)](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color)

- [ ] **Disponibilizar opções de ajuda**, como FAQ, instruções ou informações de contato acessíveis.  
  `WCAG 3.2.6 (A)` *(ITRAC)* [(9)](#)

- [ ] **Garantir design responsivo**, sem rolagem horizontal em telas pequenas.  
  `WCAG 1.4.10 (AA)` [(10)](https://www.w3.org/WAI/WCAG22/Understanding/reflow)

- [ ] **Permitir redimensionamento do texto em até 200%** sem perda de conteúdo.  
  `WCAG 1.4.4 (AA)` [(11)](https://www.w3.org/WAI/WCAG22/Understanding/resize-text)

- [ ] **Manter espaçamento adequado**:  
  - Linha ≥ 1.5x tamanho da fonte  
  - Parágrafo ≥ 2x tamanho da fonte  
  - Entre letras ≥ 0.12em  
  - Entre palavras ≥ 0.16em  
  `WCAG 1.4.12 (AA)` [(12)](https://www.w3.org/WAI/WCAG22/Understanding/text-spacing)

- [ ] **Garantir consistência de navegação** entre páginas.  
  `WCAG 3.2.3 (AA)` [(13)](https://www.w3.org/WAI/WCAG22/Understanding/consistent-navigation)

- [ ] **Garantir consistência de elementos de ajuda** ao longo do site.  
  `WCAG 3.2.6 (A)` [(14)](#)

- [ ] **Garantir tamanho mínimo de área clicável** (44x44px).  
  `WCAG 2.5.8 (AA)` [(15)](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum)

- [ ] **Informar ao usuário sua localização atual na interface** (status atual).  
  *(Heurística 1)* [(16)](https://www.nngroup.com/articles/ten-usability-heuristics/)

- [ ] **Exibir opções, ações e instruções visíveis**, evitando exigir memória do usuário.  
  *(Heurística 6)* [(17)](https://www.nngroup.com/articles/recognition-and-recall/)

- [ ] **Manter design estético e minimalista**, evitando ruído visual e informação desnecessária.  
  *(Heurística 8)* [(18)](https://www.nngroup.com/articles/ten-usability-heuristics/)

- [ ] **Evitar esconder conteúdo de tecnologias assistivas** com `display:none` ou `visibility:hidden` quando o elemento deveria ser lido.  
  *(Guia-UK)* [(19)](#)
